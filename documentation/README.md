# XMAXX Documentation

This folder explains what is deployed, how it is operated, and what XMAXX is trying to build.

## Documents

- `platform-architecture.md`
  What is running today, how traffic flows, and how the AWS, K3s, Docker, Helm, and frontend layers fit together.

- `infrastructure-and-operations.md`
  How the Terraform stacks, container workflow, Helm release model, certificate flow, and day-two operations work.

- `product-and-audiences.md`
  The product intent for `xmaxx.ai`, the design position of the company, and how different audiences should read the platform.

## Who Should Read What

- Engineers: architecture and operations
- Operators and site owners: operations and audience guidance
- Partners and vendors: product and audience guidance
- Customers and community members: product and audience guidance
- Employees and future builders: all three

## Working Rule

If the deployed system changes, update these docs in the same change set. The goal is for the repository to stay operationally truthful.
